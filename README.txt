Описание структуры файла с входными данными:
N - количество объектов
K - количество меток
После этого следует тип решаемой задачи(после запятой пробел не ставить; Правильно: 'max,+'; Неправильно: 'max, +').
Q - матрица весов вершин. Весам вершин каждого конкретного объекта соответствует столбец.
G - массив матриц весов дужек между соседними обьъектами. i-ая строка матрицы соответствует весам дужек
между i-ой вершиной одного объекта и всеми вершинами соседнего объекта. При заполнении матриц G нужно учесть, что для разных задач
при отсутствии некоторых дужек нужно по разному это указывать.
(К примеру для max,+ это вес -inf(минус бесконечность), для or,and это вес 0, для min,+ это вес inf(бесконечность)).
